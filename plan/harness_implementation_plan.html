What this plan delivers
The repository currently supports drawing multi-agent flows on a visual canvas and running them on four frameworks (LangGraph, CrewAI, Mastra, MS Agent Framework) with Langfuse observability. The 14 existing node types cover execution-level concerns — LLM calls, routing, tool invocation, HITL pauses, memory read/write, and parallelism.

The full harness (as specified in full_harness_architecture.html) adds the reasoning and control architecture that sits above those execution nodes: a world model with typed beliefs and contradiction detection, a hypothesis system with diversity enforcement, diagnostic health vectors feeding a five-tier control state resolver, VOI-gated evidence gathering, a nine-layer verification system, named recovery strategies, context compression with dependency risk tracking, an optional experience store, and a three-lens reviewer pass. These are the structures that make an AI agent reliable, not merely capable.

This plan organises the work into 11 phases that build incrementally — each phase produces a usable, testable increment rather than deferring all value to the end.

architectural invariants — all must pass before Phase 11 closes
Observation vs conclusion separationobservations[] and beliefs[] are separate world_model structures. A HIGH-reliability tool output never automatically produces a HIGH-reliability conclusion — conclusions require explicit derived_from[] links.
Normalisation contractEvery diagnostic sub-dimension entering resolve_control_state() is normalised [0,1] via a dimension-specific method. Commensurability is required for Tier 4 arithmetic.
generation_id double-increment per iterationworld_model.generation_id increments twice per loop: after sub-step A (pre-execution) and after sub-step B (post-execution). All three gates assert the matching value before reading control_state.
Deadlock detectionTier 2 of resolve_control_state() checks whether block_mask contains two entries whose recovery actions are mutually blocked. If so: escalation_reason=HUMAN_REQUIRED — no autonomous recovery attempted.
SYSTEM_BREAKING via Tier 1 onlyA SYSTEM_BREAKING contradiction never triggers an inline halt. It enters world_model.contradictions[] and is picked up by Tier 1 of the next resolve_control_state() call — same code path regardless of loop position.
control_state as sole control inputselect_best_action() reads control_state exclusively for control decisions. All other parameters (world_model, hypothesis_set, task_graph, etc.) are read-only informational context that scope the search space.
dep_class_gap advisory onlydep_class_gap_annotation is never a numeric input to any resolve_control_state() tier. It is attached to control_state.notes[] and surfaces to the adversarial reviewer and human escalation only.
Failure mode library as one of four — not a controllerfailure_mode_library contributes matched_pattern.confidence (normalised) to Tier 4 as one advisory input. It does not veto, block, or redirect execution. It is also one of four hypothesis generation sources.
Adversarial prior discarded after useadversarial_prior is seeded on causal proximity to success_criteria (not raw confidence) and discarded after adversarial_lens() completes. It never pollutes world_model.
experience_store is a no-op when absentEvery code path that uses experience_store guards with experience_store.available. Without it: warm_start is a no-op, strategy priors are flat, no cross-run learning — but the agent runs correctly.
Process Concept Invariants (P-PC)
INV-PC-01 Process concept is a strong prior, not a constraintseed_task_graph() pre-populates the task graph but does not lock it. The existing replanning machinery (diagnose_and_replan(), rebuild_task_graph()) can add, remove, or modify tasks during the run without restriction. A concept provides the starting scaffold; the harness reasoning and recovery layers own the graph from that point forward.
INV-PC-02 Concept-seeded graphs pass the same validate_task_graph() gateAfter seed_task_graph() populates the graph, validate_task_graph() and decomposition_gate() run exactly as they would for a model-driven graph. decomposition_gate() is never bypassed for concept-seeded runs. A concept whose step structure is incompatible with the objective is caught and reported via the gate response.
INV-PC-03 process_concept=None path is structurally unchangedWhen initialize_harness() is called without a process_concept (or with None), the function returns valid=True, errors=[], process_concept_id=None. No tasks are seeded. The optional field on HarnessRunState is None and does not affect any downstream logic. All pre-P-PC runs are structurally identical.
INV-PC-04 Explicitly-specified missing concept is a hard errorWhen harness_meta.process_concept_id is set in the FlowSpec but the named concept cannot be found in the registry, the run fails with ProcessConceptNotFoundError — it does not silently fall back to model-driven decomposition. An explicitly specified missing concept is a configuration error; the error response includes the missing concept ID.

Testing is organised into four tiers. Every invariant has a specific test. Integration tests run against all four frameworks.
Tier 1 — Unit tests (per-function, per-phase)
Each phase adds unit tests for every new function introduced. Tests live in adapter/tests/test_harness_pN.py. Coverage target: 90%+ for all harness-core modules. Run with pytest adapter/tests/ -v -k harness.
Tier 2 — Invariant tests (all 10 invariants)
INV-01 Observation/conclusion separation: Inject a HIGH-reliability tool observation. Assert it appears in observations[] not beliefs[]. Assert no belief is derived from it without explicit derived_from[] chain.
INV-02 Normalisation contract: For each of the 10 diagnostic sub-dimensions, inject raw values at boundary conditions (0, 1, beyond-range). Assert all outputs are clamped [0,1]. Assert TIER 4 arithmetic uses only normalised values.
INV-03 Generation-ID double-increment: Run one full loop iteration. Assert world_model.generation_id incremented exactly twice. Assert action_gate read sub-step A value, post_exec_gate read sub-step B value.
INV-04 Deadlock detection: Construct a block_mask where dep_graph_quality is critical (blocks COMMIT) and verification_strength is critical (blocks COMMIT, recovery requires dep_graph refresh). Assert escalation_reason=HUMAN_REQUIRED, not autonomous recovery.
INV-05 SYSTEM_BREAKING via Tier 1 only: Inject a SYSTEM_BREAKING contradiction mid-loop at three different positions. Assert it enters contradictions[] each time, and BLOCKED state is set only on the NEXT resolve_control_state() call — never inline.
INV-06 control_state sole control input: Instrument select_best_action(). Assert no write to control_state occurs from world_model, hypothesis_set, or task_graph directly. Assert block_mask is the only suppression mechanism.
INV-07 dep_class_gap advisory only: Assert dep_class_gap_annotation does not appear as a parameter in any tier 1–4 computation inside resolve_control_state(). Assert it is attached to control_state.notes[] only.
INV-08 Failure mode library scope: Assert failure_mode_library.match() return value enters ONLY (a) Tier 4 of resolve_control_state() and (b) hypothesis generation sources. Assert it has no write path to block_mask or escalation_reason.
INV-09 Adversarial prior discarded: Run reviewer_pass(). Assert adversarial_prior object is not referenced anywhere in world_model, hypothesis_set, or evidence_store after adversarial_lens() returns.
INV-10 experience_store no-op: Run full harness loop with experience_store.available=False. Assert outcome is identical in structure to a run with it present (no missing keys, no exceptions). Assert warm_start() returns immediately.
Tier 3 — Integration tests (per framework)
For each framework (LangGraph, CrewAI, Mastra, MAF): compile a harness flow spec to framework code, run it against a representative task, assert all 10 invariants hold at runtime. Test files: adapter/tests/test_harness_integration_LG.py, _CR.py, _MA.py, _MAF.py. Run via pytest adapter/tests/ -v -k harness_integration.
Tier 4 — End-to-end harness scenario tests
E2E-01 Happy path: Full 22-node harness loop completes, outputs contract-valid result, experience_store populated.
E2E-02 BLOCKED escalation: Inject SYSTEM_BREAKING contradiction. Assert Tier 1 fires, BLOCKED state set, escalation surfaces to HITL, caller response restarts loop.
E2E-03 Recovery cycle: Force verification failure three times on same task. Assert DIRECT_EDIT → TRACE_EXEC strategy progression. Assert global replan fires when replan_scope=GLOBAL.
E2E-04 Warm start benefit: Run harness twice on same task class. Assert second run's strategy_weights differ from flat priors. Assert warm_start loaded prior decomposition patterns.
E2E-05 Parallel branch merge: Run two parallel tasks with write-domain overlap. Assert optimistic path fires when conflict_prob ≤ PESSIMISTIC_THRESHOLD. Assert contradiction detection runs on merged world model at join.
E2E-06 Context compression: Force context_pressure.critical. Assert compress_memory() populates both compressed_structures[] and pruned_regions[]. Assert action_dep_overlap() fires correctly when a HIGH-risk action depends on a pruned region.
E2E-07 Reviewer pass re-entry: Seed reviewer with findings that reopen a completed task. Assert drain_propagation_queue() triggers loop re-entry and task moves from COMPLETE to ACTIVE.
E2E-08 max_steps budget: Set max_steps=5, run a task requiring 7 steps. Assert warning fires at step 4 (0.8×5), escalation fires at step 5 with "budget exhausted" reason.
HIGH
FlowSpec schema breakage
Adding harness node types to FlowSpec may break existing v0.2.0 flows loaded by current adapters. Mitigation: extend schema additively; all new harness fields optional with sensible defaults; version FlowSpec to v1.0.0 with migration tool.
HIGH
Diagnostic normalisation calibration
Thresholds (CRITICAL_THRESHOLD, CAUTION_THRESHOLD, PATTERN_THRESHOLD) are implementation-dependent constants per the architecture. Wrong values produce degenerate BLOCKED or NORMAL states on real tasks. Mitigation: ship calibration harness with tunable defaults; document all threshold constants prominently; treat first runs as calibration data.
HIGH
Framework adapter divergence
Harness control features (generation_id gates, 9-layer verification, adversarial reviewer) are framework-agnostic by design but must be implemented consistently across LangGraph, CrewAI, Mastra, MAF. Mitigation: implement in the adapter middleware layer, not in per-framework code; run invariant test suite against all four frameworks on each phase merge.
MEDIUM
World model unbounded growth
Observations[], beliefs[], and contradictions[] may grow without bound in long runs. Mitigation: compression policy (Phase 6) + journal_retention_policy + TTL-based staleness_sweep ensure bounded growth. Test with max_steps=1000 runs.
MEDIUM
VOI threshold calibration
VOI = expected_uncertainty_reduction × decision_impact is easy to specify but hard to calibrate. Too-low threshold causes indefinite evidence-gathering; too-high threshold skips useful evidence. Mitigation: instrument VOI decisions in Langfuse traces; provide manual override in canvas node config; start with conservative (high-VOI-required) defaults.
MEDIUM
Hypothesis diversity enforcement overhead
Forcing four structurally distinct generation sources each iteration adds latency. Mitigation: parallelise generation sources where possible; cache analogy_based_generation results; allow warm_start to seed hypothesis set to reduce full generation frequency.
LOW
Adversarial prior quality
Causal proximity filter for adversarial_prior seeding is directional but imperfect (acknowledged in the scope). Mitigation: instrument which beliefs are selected and surface to Langfuse; allow entropy-maximising seed as alternative config option.
LOW
Experience store cold start
The experience store provides no value until runs accumulate. Mitigation: ship pre-seeded experience store for common task classes; document that first N runs are calibration; make no-experience path fully correct and clearly documented.
